E-Shop Checkout UI/UX Design Guide

===========================================

VISUAL DESIGN SPECIFICATIONS

Color Palette:
- Primary Background: #f5f5f5 (light gray)
- Content Background: #ffffff (white)
- Text Primary: #333333 (dark gray)
- Text Secondary: #555555 (medium gray)
- Text Muted: #666666 (lighter gray)
- Error Text: red (CSS color name)
- Success Background: #d4edda (light green)
- Success Text: #155724 (dark green)
- Error Background: #f8d7da (light red)
- Error Border: #f5c6cb (light red border)

Button Colors:
- Pay Now Button: green (CSS color name) - REQUIRED
- Pay Now Hover: darkgreen (CSS color name)
- Pay Now Disabled: #cccccc (gray)
- Add to Cart: #28a745 (green)
- Quantity Controls: #007bff (blue)
- Remove Button: #dc3545 (red)
- Apply Discount: #6c757d (gray)

Typography:
- Font Family: Arial, sans-serif
- Body Text: 14px
- Headings: Bold, varying sizes
- Error Messages: 12px, red color
- Success Messages: 18px, bold

===========================================

LAYOUT SPECIFICATIONS

Container:
- Max Width: 800px
- Center Aligned: margin 0 auto
- Padding: 20px
- Background: White with shadow

Sections:
- Margin Bottom: 30px between sections
- Padding: 20px internal
- Border: 1px solid #dddddd
- Border Radius: 5px

Form Elements:
- Full Width: 100% for text inputs
- Padding: 10px
- Border: 1px solid #dddddd
- Border Radius: 4px
- Font Size: 14px

===========================================

ERROR HANDLING SPECIFICATIONS

Error Message Display:
- Color: red (CSS color name) - REQUIRED
- Font Size: 12px
- Position: Below respective input field
- Margin Top: 5px
- Initial State: display: none

Error Visibility Rules:
- Show Error: When field validation fails
- Hide Error: When user focuses on field OR when validation passes
- Real-time: Errors appear/disappear as user types

Specific Error Messages:
- Name Field: "Full name is required"
- Email Field: "Valid email address is required"
- Address Field: "Address is required"
- Discount Code: "Invalid or expired discount code"
- Empty Discount: "Please enter a discount code"

===========================================

INTERACTIVE ELEMENTS

Pay Now Button:
- Background Color: green (CSS color name) - MANDATORY
- Text Color: white
- Padding: 15px 30px
- Font Size: 16px
- Border Radius: 5px
- Full Width: 100%
- Disabled State: Background #cccccc, cursor not-allowed
- Hover State: Background darkgreen
- Active Condition: Enabled only when form valid AND cart not empty

Form Validation States:
- Invalid Field: Red error text visible below field
- Valid Field: No error text visible
- Focus State: Error text hidden on focus
- Real-time: Validation occurs on input/change events

Radio Button Groups:
- Margin: 10px vertical spacing
- Label Position: Inline to the right of radio button
- Default Selection: First option in group
- Visual Feedback: Standard browser radio styling

===========================================

RESPONSIVE BEHAVIOR

Mobile Considerations:
- Container: Responsive padding
- Form Elements: Full width maintained
- Button Text: No wrapping
- Cart Items: Stack on small screens
- Touch Targets: Minimum 44px height

Desktop Experience:
- Fixed Maximum Width: 800px
- Centered Layout: Auto margins
- Hover States: All interactive elements
- Keyboard Navigation: Tab order logical

===========================================

ACCESSIBILITY SPECIFICATIONS

Form Labels:
- Association: All inputs have proper label elements
- Required Fields: Marked with asterisk (*)
- Error Association: aria-describedby for error messages

Interactive Elements:
- Focus Indicators: Browser default maintained
- Color Contrast: Sufficient for all text
- Alternative Text: Not applicable (no images)

Semantic HTML:
- Form Elements: Proper input types (email, number, radio)
- Headings: Hierarchical structure (h1, h2, h3)
- Lists: Proper list markup where applicable

===========================================

ANIMATION SPECIFICATIONS

Transitions:
- Button Hover: Instant color change (no animation)
- Error Display: Instant show/hide (no fade)
- Form Updates: Immediate response
- Cart Updates: Instant DOM manipulation

Performance:
- No Heavy Animations: Maintain 60fps
- Instant Feedback: User actions trigger immediate visual response
- Smooth Scrolling: Browser default behavior
